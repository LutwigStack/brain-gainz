# 13 CS Bachelor Campaign And Daily Run

## Status

`planned`

## Goal

Use a realistic `Computer Science Bachelor` campaign as the main proving ground for BrainGainz mechanics:
- campaign templates and forks
- knowledge map structure
- routes and specializations
- mastery / XP
- assessment checks
- Daily Run sessions
- weak spot recovery
- learner vs author surfaces

The campaign can be simplified, but it must feel like a real undergraduate CS path rather than demo filler.

## Product Hypothesis

BrainGainz will only become understandable when the UI is exercised against a coherent campaign with enough depth:
- several semesters / stages
- prerequisite chains
- parallel branches
- daily tasks
- checks of different types
- weak spots
- route planning decisions

A small but realistic slice is better than a huge shallow graph.

## Campaign Direction

Working title: `Computer Science Bachelor`.

Simplified structure:
- Year 1: programming fundamentals, discrete math, calculus, computer systems intro
- Year 2: data structures, algorithms, databases, linear algebra, operating systems basics
- Year 3: software engineering, networks, compilers/programming languages, AI/ML intro
- Year 4: distributed systems, security, electives, capstone

First playable slice:
- one coherent path through `Programming Fundamentals -> Discrete Math -> Data Structures -> Algorithms`
- enough side branches to test weak spots and map overview
- at least one assessment of each supported type over time
- daily runs that can be started, completed, failed, and recovered

## Scope

Includes:
- simplified CS bachelor curriculum outline
- campaign template seed or importable fixture
- knowledge graph nodes, routes, and specializations for a first slice
- assessment metadata examples
- Daily Run session model and UI
- weak spot / recovery queue
- learner mode vs author mode boundaries
- QA pass using this campaign as the primary fixture

Excludes:
- fully accurate accreditation mapping
- full four-year curriculum completeness in the first pass
- generated art for every course
- cloud sharing / marketplace
- final AI tutor behavior

## Success Criteria

- A user can create or open a `Computer Science Bachelor` campaign template.
- The campaign has enough real structure to test map overview, route planning, Today, mastery, and assessment.
- Daily Run can produce 3-5 concrete tasks from the campaign state.
- Completing or failing a Daily Run changes visible progress/recovery state.
- Weak spots produce understandable recovery tasks.
- Author-only fields do not dominate learner-facing Today and assessment flows.
- QA can use this campaign instead of synthetic one-node examples.
