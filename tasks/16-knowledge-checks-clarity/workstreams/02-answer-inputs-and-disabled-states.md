# 02 Answer Inputs And Disabled States

## Status

`done`

## Goal

Make each check type tell the learner what kind of answer is expected and why an action is unavailable.

## Scope

- exact answer input
- number answer input
- contains answer input
- checklist input
- manual review fields
- AI-assisted review fields
- disabled submit/check/pass buttons

## Done When

- each check type has a clear input state
- disabled buttons have one nearby reason
- fields do not expose technical names in learner mode
- mobile layout remains usable
