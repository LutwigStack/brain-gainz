# 03 Attempt Results Mastery And XP

## Status

`done`

## Goal

Make check results feel like learning outcomes, not app errors.

## Scope

- successful attempt state
- failed attempt state
- retry state
- confirmed mastery display
- XP display
- difference between manual progress and confirmed progress

## Done When

- failed attempts are visually normal learning states
- successful attempts clearly show what changed
- confirmed progress and XP do not look like ordinary self-marked progress
- user can see the next action after an attempt
