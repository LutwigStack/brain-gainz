# 04 Authoring UI For Checks

## Status

`done`

## Goal

Make check authoring usable without leaking authoring complexity into learner mode.

## Scope

- type selection
- task prompt
- correct answer / criteria
- checklist items
- manual review setup
- AI-assisted review setup
- preview of what the learner will see

## Done When

- author mode clearly shows check setup fields
- learner mode hides setup fields
- check type differences are understandable
- authoring UI avoids long technical blocks by default
