# Knowledge Check Flow State Map

## Learner States

1. No verified progress yet: the learner sees the current mastery level, the confirmed progress lane, and the self-assessment lane separately.
2. Check criteria visible: the panel names the check type and says what answer format is expected.
3. Action unavailable: the readiness card gives one nearby reason, for example an empty answer or no selected checklist item.
4. Attempt submitted: local strict checks save a passed or failed attempt automatically.
5. Failed attempt: the result is shown as a normal learning attempt; progress and XP do not change.
6. Passed attempt: confirmed progress updates to the selected mastery level and XP is credited.
7. Self-assessment: the learner can mark memory without XP; it stays visually separate from confirmed progress.

## Check Types

- Exact: one answer is compared with the expected value.
- Number: a numeric answer is compared with the expected number and tolerance.
- Contains: the answer must include required terms.
- Checklist: the learner marks completed checklist items; required items decide pass/fail.
- Manual strict: a visible confirmation or technical result can confirm progress.
- AI-assisted: an AI-check result or human-entered confirmation can confirm progress.

## Author Mode

Author mode owns task prompt, criterion setup, checklist items, manual criteria, AI rubric, and preview. Learner mode sees only the check type, expected input, action readiness, attempt result, and progress effect.
