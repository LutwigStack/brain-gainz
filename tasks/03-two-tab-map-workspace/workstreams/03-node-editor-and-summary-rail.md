# 03 Node Editor And Summary Rail

## Status

`planned`

## Goal

Добавить в правый rail `Карта` две разные, но связанные секции:
- раскрывающийся `Node Editor`;
- постоянный `Node Summary`.

## Why

Если редактирование узла будет жить вне карты, `Карта` не станет главным механизмом работы. Если editor и summary слить в одну панель, исчезнет ясность ролей:
- editor отвечает за ввод и изменение данных;
- summary отвечает за состояние, прогресс и ближайшие действия.

## Scope

- collapsible `Node Editor` в верхней части rail;
- ручное редактирование основных полей узла:
  - название
  - описание
  - тип
  - статус
  - критерий завершения
  - следующий шаг
  - базовая секция связей
  - награда / результат
- постоянный `Node Summary` под editor;
- distinction между editable fields и read-oriented summary;
- подготовка flows для ручного ведения карты.

## Done When

- editor и summary одновременно существуют в одном rail и не мешают друг другу;
- редактирование узла можно выполнять, не выходя из `Карта`;
- summary по выбранному узлу остается быстрым и обзорным;
- `Карта` становится реальным рабочим экраном, а не только обзором graph state.
