# 03 Web Prototype

## Status

`done`

## Goal

Собрать первый живой web prototype на минимальной игровой оболочке.

## Scope

- экран `Сегодня`;
- карточка навыка;
- простая карта;
- один герой;
- 2–3 эффекта;
- один рабочий квестовый цикл.

## Implementation Plan

1. Проверить и добить entry flow экрана `Сегодня`, чтобы первый шаг был понятен без чтения всей карты.
2. Довести web shell до цельного прототипа: согласовать `Сегодня`, `Карту` и `Журнал` как один рабочий цикл.
3. Усилить игровой отклик после действия: герой, карта и эффекты должны заметно реагировать на выбор и завершение шага.
4. Закрыть пробелы в данных и UI карточки узла, чтобы было видно контекст, связи и следующий шаг.
5. Подтвердить workstream свежими проверками: `npm run lint`, `npm run build`, `cargo check`, `cargo test`.

## Current Notes

- В рабочем дереве уже есть основа app shell и Pixi-слоя для этого workstream.
- Перед новыми правками нужно аккуратно работать поверх существующих незакоммиченных изменений, не затирая их.

## Validation

- `npm run lint`
- `npm run build`
- `npm test`
- `cargo check --manifest-path src-tauri/Cargo.toml`
- `cargo test --manifest-path src-tauri/Cargo.toml`

## Result

- web prototype теперь запускается и без Tauri runtime;
- экран `Сегодня` показывает и проводит основной квестовый цикл;
- `Карта` синхронизируется с результатом действия и Pixi-слоем;
- `Журнал` доступен как часть общего app shell и не ломает web-поток;
- прототип ощущается как связанный продуктовый поток, а не набор отдельных экранов.

## Done When

- после открытия приложения за несколько секунд понятно, что делать сейчас;
- после выполнения действия виден игровой отклик;
- web version уже ощущается как продукт, а не как набор экранов.
