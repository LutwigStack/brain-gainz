# 03 Developer Main Campaign

## Status

`planned`

## Goal

Provide one developer-authored main campaign that can be edited from the normal interface in v1.

## Scope

- create or seed one `developer_main` campaign
- make it visible in the campaign menu
- allow editing through normal app surfaces
- use the same map, stats, and XP behavior as user campaigns
- avoid template/version/update complexity for now

## Product Direction

- The developer main campaign is the default curated experience.
- For now, it is not immutable.
- It can evolve through the UI while the project is being designed.

## Done When

- developer main campaign exists after first startup
- existing data is attached to it
- it can be opened from the campaign menu
- it can be edited like a normal campaign
- no separate template-copy flow is required in v1
