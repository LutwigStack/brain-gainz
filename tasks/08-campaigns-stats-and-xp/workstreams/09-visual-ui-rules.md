# 09 Visual UI Rules

## Status

`planned`

## Goal

Prevent the new campaign and progression surfaces from becoming text-heavy.

## Rules

- Text is a label, not the main explanation.
- Permanent UI copy should be short.
- Detailed explanation belongs in hidden panels, tooltips, or docs.
- Prefer:
  - color
  - icons
  - stat shapes
  - rings
  - progress bars
  - compact cards
  - branch chips
- Avoid:
  - long paragraphs
  - always-visible instructions
  - big tables as the first view
  - decorative cards with no action
  - multiple primary actions in one area

## Required Checks

- campaign menu can be understood visually
- wind rose is not a text dashboard
- stat drill-down is compact
- `Now` has one obvious next action
- map remains the main work surface

## Done When

- every new campaign surface passes a visual-density review
- no first-screen campaign UI depends on paragraph explanations
- labels fit their containers on desktop and mobile
- visual state carries more information than body text
