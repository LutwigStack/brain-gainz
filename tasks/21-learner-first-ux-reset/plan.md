# Learner First UX Reset Plan

## Order

1. Separate learner mode from author mode.
2. Define and implement the first session path.
3. Rebuild Today around one next action.
4. Move learner checks into a focused flow.
5. Make the map a read-only progress overview by default.
6. Clean empty states and button wording.
7. Run first 10 minutes browser QA.

## Main Risk

The team may keep adding explanations instead of simplifying the screen. Prefer structure, state, and action hierarchy over more text.

## UX Rule

For each screen, write down:

- where am I?
- what is the main thing to do?
- what happens if I press the main button?
- what is secondary?
- what is hidden unless I am editing?

If the answer is unclear, the screen is not done.

## QA Targets

- first app open
- campaign selection
- Today empty state
- Today with available lesson
- Start lesson
- answer check
- failed answer
- passed answer
- confirmed progress and XP
- next step after result
- map as learner overview
- author/edit mode entry and exit
- desktop 1280px
- mobile 390px
