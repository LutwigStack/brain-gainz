# Russian CS Content Glossary

## Product Terms

| English source | Russian UI term | Notes |
| --- | --- | --- |
| Computer Science | информатика | Used in campaign/domain names: "Бакалавриат по информатике", "Основы информатики". |
| campaign | кампания | App-level term from the shell; course content uses the campaign name directly. |
| route | маршрут | Used for the learning path. |
| stage | этап | Stage names are content titles such as "Алгоритмы" or "Базы данных". |
| node | узел | Kept for map/graph learning units. |
| check | проверка | Used for answer validation and task checks. |
| attempt | попытка | Used for check attempts and latest attempt copy. |
| confirmed progress | подтвержденный прогресс | Tied to verified mastery and completion. |
| manual progress | ручное подтверждение | Used when a learner or reviewer confirms completion. |
| weak spot | слабое место | Used for recovery and practice selection. |
| Daily Run | дневной заход | The app may keep the product label, but task copy should read naturally in Russian. |
| Wind Rose | роза ветров | Product surface name can remain if branded; branch text is Russian. |

## Kept Technical Forms

SQL, CRUD, ACID, ORM, Big-O, NULL, SELECT, WHERE, GROUP BY, LIFO stay as technical terms or syntax. Russian explanations around them should be complete enough that the learner does not need English instructions.
