# Focused Lesson And Quiet Today Plan

## Order

1. Make lesson/check own the screen during learning.
2. Reduce Today visual noise around the main action.
3. Make mobile navigation clearer before icon-only use.
4. Clean task statuses and QA artifact consistency.
5. Run desktop and mobile browser QA.

## Main Risk

Do not hide useful features by breaking access. Demote them from the first path instead.

## UX Checks

For Today:
- Is the current lesson the first thing the eye understands?
- Is there exactly one dominant action?
- Are game/progress/detail panels visually secondary?

For Lesson:
- Is the question/task central?
- Is the answer/check action central?
- Is the result clear?
- Is the map optional, not a competing workspace?

For Mobile:
- Can a new user understand the first path without memorizing icons?
- Is the main action visible without excessive scrolling?
- Is there no horizontal overflow?

## QA Targets

- campaign template selection
- Today before first lesson
- direct lesson/check screen
- failed checklist attempt
- passed checklist attempt
- next step after pass
- map opened as secondary action
- mobile Today
- mobile lesson/check
- mobile nav labels/states
