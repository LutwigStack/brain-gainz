# 02 Lesson Header Compression

## Status

`planned`

## Goal

Make the top of the lesson short and useful.

The learner should not see several metadata boxes before understanding the task.

## Scope

- reduce the current lesson header to the minimum needed context
- keep title, short instruction, and route context
- remove or collapse numbered metadata tiles if they do not help the learner
- make `Обзор карты` and `Сегодня` secondary actions
- keep result summary visible after completion

## Done When

- first viewport shows the lesson title, task, criteria, and main action faster
- no dense metadata row dominates the top of the lesson
- desktop and mobile layouts remain stable
