# 17 CS Course Second Slice

## Status

`done`

## Goal

Add a second realistic learning block to the Computer Science campaign so BrainGainz is tested on more than the first foundations route.

The first slice proved the loop on Programming Fundamentals, Discrete Math, Data Structures, and Algorithms. The next slice should add enough real course structure to test route choice, daily tasks, checks, weak spots, map overview, and statistics at a larger scale.

## Candidate Blocks

Choose one for the next slice:
- databases
- operating systems
- networks
- software engineering
- introductory AI

Recommended first choice: databases.

Why databases:
- understandable to users
- good fit for exact/contains/checklist checks
- natural practical tasks
- useful route structure without needing a huge simulation
- connects well to programming fundamentals and data structures

## Scope

Includes:
- curriculum outline for the chosen block
- 25-40 nodes
- route stages
- prerequisites from the existing CS slice
- several check types
- daily task data
- weak spot recovery opportunities
- route overview and Wind Rose coverage

Excludes:
- full four-year CS curriculum
- final academic accreditation accuracy
- generated art for every topic
- full SQL execution environment
- AI tutor behavior

## Success Criteria

- The new block feels like a real course section, not filler.
- The map stays readable.
- Today can select tasks from both the existing and new slice.
- Checks include at least three types.
- Weak spots and recovery work with the new content.
- Route overview and statistics do not become noisy.
- Browser QA proves daily use on the larger campaign.
