# CS Course Second Slice Outline

## Decision

Use databases as the second Computer Science course block.

## Route Shape

The CS route now has five required stages:

- Programming Fundamentals: 12 nodes
- Discrete Math For CS: 10 nodes
- Data Structures: 12 nodes
- Algorithms: 10 nodes
- Database Systems: 28 nodes

Total route size: 72 required nodes.

## Prerequisites

The first database node depends on completed programming and data-structure foundations:

- Small Program Design
- Data Structure Tradeoffs

Later database nodes connect to existing CS foundations:

- Hash Tables and Asymptotic Complexity for indexes
- Tracing State By Hand for isolation and race conditions
- Basic Unit Tests and Debugging A Small Program for migrations and database testing
- Input Output And Parsing for SQL injection
- Memory Model for embedded versus server database tradeoffs

## Check Mix

The slice uses six check methods:

- checklist
- contains
- exact
- number
- LLM rubric
- manual strict

This keeps simple database facts strict while leaving design and tradeoff questions open enough for real explanations.

## Today Behavior

Today now promotes the active route front when that node has an available action. This keeps the route as the main daily learning flow when non-route support tasks are also available. A regression test confirms that after the first route slice is completed, Data Modeling Purpose becomes the primary Today recommendation and Database Systems becomes the current stage.
