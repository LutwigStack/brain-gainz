# 04 Mobile Result Compactness

## Status

`planned`

## Goal

Make pass/fail result states compact on 390px mobile.

## Scope

- reduce repeated form content after result
- keep result and next action near each other
- keep criteria visible but not overwhelming
- check button and secondary actions wrapping
- verify no horizontal overflow

## Done When

- pass result and `Следующий шаг` are easy to find on mobile
- failed result and retry are easy to find on mobile
- screenshots prove both states
