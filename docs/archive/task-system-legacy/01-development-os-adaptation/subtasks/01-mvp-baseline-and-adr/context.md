# Subtask Context: MVP Baseline And ADR

## Goal

Lock the approved MVP product decisions into repo-local task artifacts and ADR surfaces so later implementation work inherits a stable baseline.

## Inputs

- Epic-level product decisions for decay, daily session, cards, projects, barriers, errors, explainability, and meaningful day.
- Existing planning artifacts under the parent epic.

## Delivered Outcome

- Epic docs capture the approved MVP baseline.
- Repo-local ADR records the same decision package as an accepted architectural/product contract.
