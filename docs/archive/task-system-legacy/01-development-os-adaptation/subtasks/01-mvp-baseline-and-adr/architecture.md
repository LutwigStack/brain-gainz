# Subtask Architecture Notes: MVP Baseline And ADR

## Responsibility

- Freeze the MVP baseline at the epic level.
- Ensure the same decision package exists in both task docs and ADR form.

## Boundaries

- No product code.
- No schema implementation.
- No recommendation formula beyond documented MVP rules.

## Output Surface

- Parent epic remains the source of shared architecture and planning truth.
- ADR is the durable accepted decision record.
